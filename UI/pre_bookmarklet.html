Understanding Bookmarklets

Master the art of browser automation with JavaScript-powered bookmarks

Introduction

A bookmarklet is a small piece of JavaScript code saved inside a browser bookmark. When clicked, it executes directly on the current webpage, allowing you to automate tasks, modify pages, create quick tools, and understand how JavaScript interacts with the Document Object Model (DOM).

Think of bookmarklets as portable mini-applications that live in your browser's bookmark bar, ready to enhance any webpage you visit.

What You Will Learn

  • What bookmarklets are and how they work under the hood
  • JavaScript fundamentals including functions and DOM manipulation
  • How the javascript: URL scheme enables code execution
  • Best practices for writing secure and efficient bookmarklets
  • Real-world applications and advanced techniques

Key Concepts

What Is a Bookmarklet?

A bookmarklet is simply a bookmark that contains JavaScript code instead of a regular URL. The special javascript: protocol tells the browser to execute the code rather than navigate to a webpage.

Basic Format: javascript:alert('Hello, World!')

DOM Manipulation

Bookmarklets can access and modify the current page's Document Object Model (DOM). This allows you to change styles, content, structure, and behavior of any webpage.

Example: document.body.style.background = '#0f172a';

URL Encoding

Complex bookmarklets may need URL encoding to preserve special characters. Modern browsers handle this automatically, but understanding encoding helps debug issues.

Pro Tip Use online URL encoders or JavaScript's encodeURIComponent() function.

Security Considerations

Modern browsers implement security restrictions to protect users. Bookmarklets may be blocked on certain sites (banking, government) or when attempting sensitive operations like accessing cookies or local storage.

Important: Only use bookmarklets from trusted sources to avoid malicious code execution.

Code Examples

Example 1: Change Page Background 
javascript:document.body.style.background='linear-gradient(135deg, #667eea 0%, #764ba2 100%)';
Example 2: List All Page Links 
javascript:alert([...document.links].map(l => l.href).join('\n'));
Example 3: Show All Images 
javascript:document.querySelectorAll('img').forEach(img => console.log(img.src));
Example 4: Make Page Editable 
javascript:document.body.contentEditable='true'; document.designMode='on'; void 0;

// Particle animation const canvas = document.getElementById('particles-bg'); const ctx = canvas.getContext('2d'); const particles = []; const particleCount = 100; function setCanvasSize() { canvas.width = window.innerWidth; canvas.height = window.innerHeight; } setCanvasSize(); window.addEventListener('resize', setCanvasSize); class Particle { constructor() { this.x = Math.random() * canvas.width; this.y = Math.random() * canvas.height; this.vx = Math.random() * 0.5 - 0.25; this.vy = Math.random() * 0.5 - 0.25; this.size = Math.random() * 2; this.color = Math.random() < 0.5 ? '#22c55e' : '#10b981'; } update() { this.x += this.vx; this.y += this.vy; if (this.x < 0) this.x = canvas.width; if (this.x > canvas.width) this.x = 0; if (this.y < 0) this.y = canvas.height; if (this.y > canvas.height) this.y = 0; } draw() { ctx.beginPath(); ctx.arc(this.x, this.y, this.size, 0, Math.PI * 2); ctx.fillStyle = this.color; ctx.fill(); } } for (let i = 0; i < particleCount; i++) { particles.push(new Particle()); } function animate() { ctx.clearRect(0, 0, canvas.width, canvas.height); particles.forEach(particle => { particle.update(); particle.draw(); }); particles.forEach((p1, i) => { particles.slice(i + 1).forEach(p2 => { const dx = p1.x - p2.x; const dy = p1.y - p2.y; const distance = Math.sqrt(dx * dx + dy * dy); if (distance < 150) { ctx.beginPath(); ctx.moveTo(p1.x, p1.y); ctx.lineTo(p2.x, p2.y); ctx.strokeStyle = `rgba(34, 197, 94, ${0.1 * (1 - distance/150)})`; ctx.stroke(); } }); }); requestAnimationFrame(animate); } animate();
