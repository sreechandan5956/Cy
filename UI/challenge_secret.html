Web Authentication & Token Security

Master modern authentication mechanisms, token security, and common vulnerabilities in web applications

Introduction

Modern web applications use various methods to authenticate users and maintain sessions. Understanding how these authentication mechanisms work - and their potential vulnerabilities - is crucial for both developers and security professionals.

This comprehensive guide covers fundamental concepts in web authentication, token-based systems, JWT security, and common vulnerabilities you'll encounter when testing web applications.

What You Will Learn

  • How to inspect web applications for information disclosure
  • Understanding the difference between encoding and encryption
  • Token-based authentication and how it works
  • JWT structure, components, and security principles
  • Common authentication vulnerabilities and exploits
  • Using browser developer tools for security analysis
  • Ethical security testing methodology and best practices

Key Concepts

HTML Source Code Inspection

Developers sometimes leave sensitive information in HTML comments or hidden fields. The page source shows the raw HTML sent by the server, which may contain information not visible in the rendered page.

  • HTML comments use <!-- comment --> syntax and are ignored by browsers
  • Hidden input fields have type="hidden" but are visible in source
  • JavaScript files may contain hardcoded credentials or API keys
  • Configuration files can expose sensitive endpoints or settings
How to View Source: Right-click anywhere on the page → "View Page Source" or press Ctrl+U (Windows/Linux) or Cmd+Option+U (Mac)
Base64 Encoding

Base64 is a method to encode binary data into ASCII text. It's NOT encryption - it's just encoding! Anyone can decode it without a key.

Characteristics of Base64:

  • Uses only A-Z, a-z, 0-9, +, / characters
  • Often ends with = or == for padding
  • Length is always a multiple of 4
  • Easily recognizable pattern in strings

Common Tools base64decode.org, CyberChef, or command line: echo "data" | base64 -d

JWT (JSON Web Tokens)

JWT is a compact, URL-safe way to securely transmit information between parties. It's commonly used for authentication in modern web applications and APIs.

JWT Structure 
header.payload.signature
(Base64).(Base64).(Cryptographic Signature)

Example:
eyJhbGc.eyJ1c2Vy.SflKxwRJ

Header

Specifies the algorithm (e.g., HS256, RS256) and token type (JWT)

Payload

Contains claims - user data, roles, permissions, expiration time

Signature

Created using header + payload + secret key. Prevents tampering

Critical Understanding: The header and payload are only Base64 encoded, NOT encrypted. Anyone can decode and read them. The signature is what provides security by verifying the token hasn't been tampered with.
JWT Security Concepts

JWT security relies entirely on the secret key used for signing:

  • Without the secret key, you cannot create valid signatures
  • If the secret is weak (e.g., "secret", "password"), it can be cracked through brute force
  • Once you have the secret, you can forge any token with any claims
  • The signature algorithm matters (HS256 vs RS256 have different security models)
  • Token expiration (exp claim) limits the damage from stolen tokens
Browser Developer Tools

Modern browsers include powerful built-in tools for inspecting web applications:

Elements/Inspector

View and modify HTML/CSS in real-time

Console

Execute JavaScript and view errors

Network

Monitor HTTP requests and responses

Application/Storage

View cookies, localStorage, sessionStorage

Access Press F12 or right-click → "Inspect" / "Inspect Element"

Privilege Escalation

Privilege escalation is the act of gaining higher access permissions than originally granted. This vulnerability occurs when:

  • User-controlled data determines permissions (like modifiable JWT payloads)
  • Access controls are only enforced client-side, not server-side
  • Tokens can be forged or manipulated without detection
  • Role or permission checks are insufficient or bypassable
  • Horizontal escalation: accessing other users' data at the same privilege level
  • Vertical escalation: gaining admin/elevated access from standard user role

Essential Tools for Web Security Testing

Browser Developer Tools

Built into all modern browsers - your first line of defense for web security testing:

  • Elements: Inspect and modify HTML/CSS structure in real-time
  • Console: Execute JavaScript commands and view error messages
  • Network: Monitor all HTTP/HTTPS requests and responses with full headers
  • Application/Storage: View and modify cookies, localStorage, sessionStorage, IndexedDB
  • Sources: Browse, debug, and set breakpoints in JavaScript files
Encoding/Decoding Tools
  • CyberChef: Swiss-army knife for data transformation - gchq.github.io/CyberChef
  • Command Line: Built-in tools like base64, openssl, echo
  • Online Decoders: Quick web-based tools for Base64, URL encoding, hex conversion
  • Browser Console: atob() for Base64 decode, btoa() for encode
Token Analysis Tools
  • jwt.io: Decode, verify, and generate JWT tokens with visual interface
  • jwt_tool: Python tool for comprehensive JWT testing and exploitation
  • Burp Suite: Comprehensive web security testing platform with JWT extensions
  • OWASP ZAP: Open-source web application security scanner
Cryptographic Tools
  • hashcat: GPU-accelerated password and hash cracking
  • John the Ripper: Multi-platform password cracking utility
  • OpenSSL: Toolkit for cryptographic operations and SSL/TLS testing
  • jwt-cracker: Specialized tool for cracking JWT secrets

Security Testing Methodology

Reconnaissance Phase

The first step in any security assessment is gathering information about the target:

  • Examine visible functionality and user interfaces thoroughly
  • Review all client-side code (HTML, CSS, JavaScript files)
  • Identify technologies, frameworks, and libraries in use
  • Map out application structure, endpoints, and API calls
  • Look for comments, debugging code, or configuration files
